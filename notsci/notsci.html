这个系列参考的是 xkcd 的 what-if 模式，不过互动性更强：

  1. 收集话题

    作品的话题将从果壳问答中选择，我主要关注的是 物理学 ， 科幻 和 what-if 标签（ LePtC 标签为网友按照自己喜好创建的，不建议用于投递问题。）

  2. 构思剧本

    由于目的不再是回答题主的问题，而是创作我自己喜欢的作品，所以可能会为情节需要扩大或调整原题目。

  3. 制作初稿

    查资料，推导，计算，排版，绘制插图草图。

  4. 同行评议

    初稿完成后会挂在 Github 主页上，然后邀请果壳相关领域达人对稿件进行审稿，也欢迎相关专业的童鞋自愿前来挑错～

稿件有 3 种可能的结果：

  1. 审稿人提出修改意见：改稿
  2. 在最后一次修改之后的一周时间，没有人提出新的修改意见：画完插图，发稿
  3. 文章从出发点就存在根本问题，且无改善的可能或必要：撤稿
